Clean Code / Coding Conventions

Here I have written some of the coding conventions I find worthy of mentioning.
Clean code is important to maintain readable and less error-prone code.
It also makes maintenance easier and provides better quality.

Principles:
  • • Utilize proper naming conventions - camel case for example.
  • • Avoid deep nesting, for example: loops.
  • • Comment your code
  • • Avoid large functions - split them up.
  • • Avoid code repetition.
  • • Descriptive function names over concise.
  • • Be consistent with verbs, eg. dont use fetchUser and getUser. Only one.
  • • Capitalize constants.
  • • Unit Testing
  • • Prefer polymorphism to if/else or switch/case.
  • • KISS Principle.
  • • Avoid logical dependency.
    Don't write methods which works correctly depending on something else in the same class.
  • • Last but least, follow standard conventions of the tool/language you are using.
